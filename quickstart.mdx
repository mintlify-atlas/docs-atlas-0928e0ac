---
title: "Quickstart: run Ask Nova API and send your first chat"
description: "Install dependencies, set environment variables, add documents to the knowledge base, and send your first chat request to Ask Nova in five steps."
---

Ask Nova runs as a local FastAPI server. Once you have your API keys and at least one document in the knowledge base, the server indexes your content on startup and is ready to answer questions.

## Prerequisites

Before you begin, make sure you have:

- **Python 3.9 or later** — check with `python --version`
- **pip** — bundled with most Python installations
- A **Groq API key** — get one at [console.groq.com](https://console.groq.com)
- A **HuggingFace API key** — get one at [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)

<Steps>
  <Step title="Clone the repository and install dependencies">
    Clone the repository to your machine, then install all Python dependencies:

    ```bash
    pip install -r requirements.txt
    ```

    The key packages this installs are `fastapi`, `uvicorn`, `agno`, `lancedb`, `sentence-transformers`, and `groq`. Full versions are pinned in `requirements.txt`.
  </Step>

  <Step title="Create a .env file with your API keys">
    In the same directory as `app.py`, create a `.env` file with the following variables:

    ```bash
    HF_TOKEN=your_huggingface_token
    HUGGINGFACE_API_KEY=your_huggingface_token
    GROQ_API_KEY=your_groq_api_key
    ```

    <Note>
      `HF_TOKEN` and `HUGGINGFACE_API_KEY` can be the same value. The server raises a `RuntimeError` on startup if either is missing.
    </Note>

    To run on a non-default port, add:

    ```bash
    PORT=9000
    ```
  </Step>

  <Step title="Create the docs/ folder and add documents">
    Ask Nova answers questions from the files in the `docs/` directory next to `app.py`. Create the folder and add at least one file:

    ```bash
    mkdir docs
    ```

    Then copy your PDF, TXT, or Markdown files into it:

    ```bash
    cp your-manual.pdf docs/
    cp faq.md docs/
    ```

    <Tip>
      You can add files in sub-folders too — the server reads `docs/` recursively. Supported formats are `.pdf`, `.txt`, and `.md`.
    </Tip>

    The server indexes all documents automatically on startup. You can also trigger a re-index at any time without restarting — see [Adding documents](/guides/adding-documents).
  </Step>

  <Step title="Start the server">
    Run the development server with auto-reload:

    ```bash
    uvicorn app:app --reload --host 0.0.0.0 --port 8000
    ```

    On startup you should see log lines like:

    ```
    Indexing PDF: /path/to/docs/your-manual.pdf
    Ask Nova indexed 2 files on startup.
    INFO:     Uvicorn running on http://0.0.0.0:8000
    ```

    <Warning>
      If the `docs/` folder is empty or missing, startup indexing is skipped and Ask Nova will have no knowledge to draw from. It will still respond to `/chat` requests but will say the answer is not in the knowledge base.
    </Warning>
  </Step>

  <Step title="Send your first chat request">
    With the server running, send a test message using curl:

    ```bash
    curl -X POST http://localhost:8000/chat \
      -H "Content-Type: application/json" \
      -d '{"message": "What is SoftSUITE Novo?"}'
    ```

    You should receive a response like:

    ```json
    {
      "session_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "answer": "SoftSUITE Novo is an HR management platform..."
    }
    ```

    To continue the conversation, pass the `session_id` from the previous response back in your next request:

    ```bash
    curl -X POST http://localhost:8000/chat \
      -H "Content-Type: application/json" \
      -d '{
        "message": "How do I reset my password?",
        "session_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
      }'
    ```

    The API keeps the last three conversation turns in context, so follow-up questions work as expected. Session history is stored in SQLite at `.api_agent_data/sessions.db`.
  </Step>
</Steps>
