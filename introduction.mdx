Ask Nova is the AI support backend for SoftSUITE Novo. It exposes a simple REST API that accepts plain-text questions and returns accurate, document-grounded answers — no hallucinations, no invented workflows. Every response is drawn from the official documentation you provide, making it safe to deploy as a customer-facing support assistant.

## How the RAG pipeline works

Ask Nova uses Retrieval-Augmented Generation (RAG) to connect user questions to your document library before the language model ever produces a word.

When a request arrives at `POST /chat`, the pipeline runs in four stages:

1. **Embed the question** — the HuggingFace `sentence-transformers/all-MiniLM-L6-v2` model converts the user's message into a 384-dimensional vector.
2. **Retrieve relevant chunks** — LanceDB runs a hybrid search (dense vector + full-text) against the indexed document corpus and returns the most relevant 500-token chunks.
3. **Build context** — the retrieved chunks and the last three conversation turns are assembled into a prompt.
4. **Generate an answer** — Groq's `llama-3.3-70b-versatile` model (temperature 0.1, max 2 048 tokens) produces a grounded, conversational response.

The answer and the `session_id` are returned to the caller. Pass the same `session_id` on the next request to continue the conversation.

## Architecture overview

Ask Nova is a single-file FastAPI application (`app.py`) with no external services beyond the Groq and HuggingFace APIs.

| Component | Technology | Location |
|---|---|---|
| API framework | FastAPI 0.135 | `app.py` |
| Embeddings | HuggingFace `all-MiniLM-L6-v2` | HuggingFace Inference API |
| Vector store | LanceDB 0.30 | `.api_agent_data/lancedb` |
| LLM | Groq `llama-3.3-70b-versatile` | Groq API |
| Session storage | SQLite | `.api_agent_data/sessions.db` |
| Documents | PDF, TXT, MD | `docs/` |

<Note>
  Ask Nova only answers from documents in the `docs/` folder. If the knowledge base is empty or the answer is not supported by the retrieved chunks, the assistant says so explicitly rather than guessing.
</Note>
